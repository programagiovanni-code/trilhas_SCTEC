1. Descrição e Objetivos do Projeto
Este projeto consiste no desenvolvimento de uma landing page funcional e responsiva, destinada à apresentação do meu portfólio pessoal e profissional. A aplicação foi desenvolvida como desafio e atividade prática extra da trilha Primeiros Passos referente ao curso de Introdução à Programação Front-End e Back-End do programa SCTEC.

O objetivo principal desta página é apresentar em um único espaço digital a minha trajetória de transição para a "Carreira Tech", evidenciando minha capacidade de nir conhecimentos interdisciplinares — que vão desde ua área da saúde até rotinas administrativas e de tecnologia — para a resolução de problemas reais.

2. Estrutura da Página Web
A página foi construída atendendo aos requisitos do edital, contendo mais de três seções fundamentais:
Descrição (Sobre Mim): Apresentação pessoal, destacando meu nome (Giovanni), minha busca por uma atuação com propósito e o link para meu currículo Lattes com meu histórico acadêmico completo.
Habilidades: Uma divisão clara entre competências técnicas (Linguagens de Marcação, Programação e Inteligência Artificial) e habilidades comportamentais (como aprendizado contínuo, pensamento analítico e adaptabilidade).
Projetos: Uma vitrine com os meus principais trabalhos práticos e estudos de caso.
Missão e Contatos: Uma seção dedicada aos meus objetivos de vida e canais de comunicação direta (com links mailto: e tel:), além de um formulário interativo de feedback para posterior implementação.

3. Lista de Projetos Apresentados
Para demonstrar minha aplicação prática da tecnologia, os seguintes projetos foram documentados no portfólio:
3.1. Estudo de Caso Meteorológico: Tornados de 07/11/25
Descrição: Análise aprofundada de dados meteorológicos para compreender a dinâmica e a formação de sistemas severos na região Sul do Brasil, cruzando variáveis atmosféricas complexas.
Tecnologias utilizadas: Python e biblioteca Pandas para manipulação de dados.
Link da demonstração: Projeto de pesquisa interno (Executado localmente).
3.2. Revisão Sistemática: Chuvas intensas na Grande Florianópolis:
Descrição: Análise aprofundada com mais de 30 anos de dados meteorológicos para compreender a dinâmica e a formação de eventos de chuva extrema, buscando possiveis correlações e insights com outros dados meteorológicos para auxiliar na previsão destes eventos.
Link da demonstração: Projeto de pesquisa interno (Executado localmente).
3.3. Dashboard de Análise do Mercado Financeiro
Descrição: Ferramenta desenvolvida para monitorar e analisar indicadores financeiros, cotações de mini-índice, mini-dólar e títulos do Tesouro Direto, visando facilitar a tomada de decisão no day trade e investimentos de longo prazo.
Tecnologias utilizadas: Python e framework Streamlit.
Link da demonstração: Projeto em fase de testes locais.
3.4. Landing Page de Portfólio Pessoal
Descrição: O projeto atual. Desenvolvimento integral de uma página web responsiva e interativa para apresentar meu percurso formativo.
Tecnologias utilizadas: HTML5, CSS3 e JavaScript.
Link da demonstração: O projeto roda abrindo o arquivo index.html em qualquer navegador web.
3.5. Análise Exploratória de Dados: Sobrevivência no Titanic
Descrição: Projeto de introdução a Data Science focado na realização de uma Análise Exploratória de Dados (AED) utilizando a base de dados pública do Titanic em formato CSV. O processo envolveu a importação, tratamento de valores nulos e organização das colunas, seguido pela aplicação de agrupamentos (GroupBy) e construção de visualizações gráficas para identificar padrões e fatores associados à sobrevivência dos passageiros.
Tecnologias utilizadas: Python, Fonte de Dados e Visualizações Geradas.
Link da demonstração:Projeto de curso (Executado localmente).
3.6. Análise Exploratória: Uber Ride Analytics
Descrição: Análise exploratória utilizando o conjunto de dados da Uber ("uber-ride-analytics-dashboard") disponibilizado no Kaggle. O objetivo principal da análise foi compreender padrões de utilização do serviço de transporte urbano e suas variações temporais. Essa compreensão permite preparar e estruturar os dados para que, no futuro, possam apoiar a aplicação de modelos introdutórios de Inteligência Artificial (como tarefas de classificação ou regressão).
Tecnologias utilizadas: Python, Fonte de Dados e Visualizações Geradas.
Link da demonstração: Projeto de curso (Executado localmente).

4. Tecnologias, Arquivos e Funcionalidades
Para a construção desta aplicação, e visando atingir a pontuação máxima nos critérios de codificação , foram utilizadas as linguagens HTML, CSS e JavaScript. A entrega é composta pelos seguintes arquivos:
index.html: Responsável por toda a estrutura semântica da página, abrigando as seções de perfil, habilidades, projetos e formulário de contato.
style.css: Folha de estilos que garante o design visual, a organização em blocos (utilizando Flexbox) e a responsividade do layout.
script.js: Arquivo responsável pela interatividade da página, especificamente escutando o evento de submissão do formulário de contato e gerando um alerta na tela para o usuário, validando o uso de lógica de programação no lado do cliente.
README.md: Este arquivo de documentação, que detalha o escopo e o funcionamento do projeto.
